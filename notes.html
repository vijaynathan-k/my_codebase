CMake : Hello world

Basic Project Structure

Create a folder for your project. A standard minimal setup looks like this:

Plaintext

my_project/
├── CMakeLists.txt # The “instructions” file
└── main.cpp # Your source code

The CMakeLists.txt file
This file tells CMake what to build. Copy this into yours:

CMake
cmake_minimum_required(VERSION 3.10)
project(HelloWorld)
#Create an executable named ‘my_app’ from ‘main.cpp’
add_executable(my_app main.cpp)

The Build Workflow (Out-of-Source)

The best practice is to build in a separate folder so you don’t clutter your source code with temporary files.

Open your terminal in the my_project folder.

Create a build directory:

Bash

mkdir build && cd build

Configure the project:

Bash

cmake …
(This checks for your compiler and creates a Makefile or Visual Studio project.)

Build the project:

Bash

cmake --build .

Run it:

Linux/Mac: ./my_app
Windows: .\Debug\my_app.exe

IDE Integration (Easier Way)

If you don’t want to use the command line, modern IDEs handle this automatically:

VS Code: Install the “CMake Tools” extension. It will detect your CMakeLists.txt and provide “Build” and “Run” buttons at the bottom.
